模块

模块维护方便，增加代码重复利用，当做命名空间使用，避免命名冲突

如何使用模块？ 模块直接导入

语法：

import module_name
module_name.fun_name
module_name.class_name

当module_name 开头为数字时，可以借助importlib

语法：

import importlibt
ex=importlib.import_module("01") #01.py为文件
stu=tuling.Student()

模块名可以用别名代替：import 模块 as 别名

用模块中的部分小块：(没有前缀)

from module_name import func_name,class_name
ex=class_name()
func_name()

导入模块所有内容from module_name import * （没有前缀）

这与import module_name 导入内容一样，但有区别。

if name = “main“ 程序入口

可以有效避免模块代码被导入的时候被动执行的问题

模块的搜索路径和存储
系统默认的模块搜索路径
import sys
sys.path 属性可以获取路径列表

添加搜索路径: sys.path.append(dir)

模块加载顺序
1.上搜索内存中已经加载好的模块
2.搜索python的内置模块
3.搜索sys.path路径

包

包是一种组织管理代码的方式，包里面存放的就是模块文件

包的结构：

__init__.py
 module1.py
 module2.py
...

导入包中init.py的内容：import package_name

导入包中的模块：import package_name.module_name

导入指定init.py内容：from package_name import *

导入包中指定的模块的所有内容：from package_name.module_name import *

all的用法

在使用from package import * ,导入的实际上是init.py里面的内容。

但是当有all的时候，则按照all指定的子包或者模块进行。

例如：all=[‘module1’,’module2’,’package1’,….]

命名空间

用于区分不同位置不同功能但相同名称的函数或者变量的一个特定前缀，防止命名冲突

例如 

setName()
Dog.setName()
Person.setName()

异常

try 
except 
finally
手动抛出异常 raise

自定义异常必须是系统异常的子类

常用模块

1.calendar：获取一年的日历字符串

cal=calendar.calendar(2018)

参数

w=每个日期之间的间隔字符数 
l=每周所占用的行数
c=每个月之间的间隔字符数
cal=calendar.calendar(2018,l=0,c=5)

isleap:判断某一年是否为闰年

calendar.isleap(2018)

leapdays：获取指定年份之间的闰年的个数

calendar.leapdays(2000,2018)

month() 获取某个月的日历字符串

m=calendar.month(2018,7)

monthrange()获取一个月从周几开始和总天数

返回元组(周几，总天数) 周几（0~6）

w,t=calendar.monthrange(2017,3)
print(w)
print(t)

time模块

时间戳

time.time()

一个时间表示，从1970，01，01，0h，0min,0s到现在经历的秒数

32位操作系统能够支持到2038年

UTC时间-世界协调世界

中国时间是UTC+8 东八区

夏令时

#####时间元组
一个包含时间内容的普通元组

#####时间模块属性
timezone：当前时区和UTC时间相差的秒数

localtime：得到当前时间的时间结构

asctime()：返回元组字符串化之后的时间格式

格式：time.asctime(时间元组)

例如 

t=time.localtime()
tt=time.asctime(t)

clock：获取cpu时间（py3.0-py3.3）

sleep内单位为s

strftime：将时间元组转化为自定义字符串格式
